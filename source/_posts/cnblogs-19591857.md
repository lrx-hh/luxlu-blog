<pre><code class="language-plain">²:0xB2: 1 0 1 1  0 0 1 0

</code></pre>
<p>仅 1 比特差异！</p>
<p>因此，<code>²</code> 很可能是 <code>'2'</code> 因最高位被翻转（0→1）而变成 <code>0xB2</code></p>

<p><font style="color: rgba(51, 51, 51, 1)">某互联网大厂的安全组件V2.0引入了"双重保护机制"来保护敏感数据：</font></p>

<h4 id="21-加密流程梳理"><font style="color: rgba(51, 51, 51, 1)">2.1 加密流程梳理</font></h4>
<p><font style="color: rgba(51, 51, 51, 1)">完整的加密流程如下：</font></p>
<pre><code class="language-plain">原始手机号 → PBKDF2派生密钥 → AES-128-CBC加密 → 十六进制编码 → 字符映射混淆 → 最终密文
</code></pre>
<p><font style="color: rgba(51, 51, 51, 1)">具体步骤：</font></p>
<ol>
<li><font style="color: rgba(51, 51, 51, 1)">使用</font><font style="color: rgba(51, 51, 51, 1)"> </font><code>&lt;font style="color:rgb(192, 52, 29);background-color:rgb(246, 246, 246);"&gt;PBKDF2(user_id, SALT)&lt;/font&gt;</code><font style="color: rgba(51, 51, 51, 1)"> </font><font style="color: rgba(51, 51, 51, 1)">生成16字节AES密钥</font></li>
<li><font style="color: rgba(51, 51, 51, 1)">使用AES-128-CBC模式（IV固定为</font><font style="color: rgba(51, 51, 51, 1)"> </font><code>&lt;font style="color:rgb(192, 52, 29);background-color:rgb(246, 246, 246);"&gt;b"Dynamic_IV_2026!"&lt;/font&gt;</code><font style="color: rgba(51, 51, 51, 1)">）加密手机号</font></li>
<li><font style="color: rgba(51, 51, 51, 1)">将加密结果转换为十六进制字符串</font></li>
<li><font style="color: rgba(51, 51, 51, 1)">对十六进制字符串的每个字符应用自定义映射表进行混淆</font></li>
</ol>

<h3 id="三解题思路"><font style="color: rgba(0, 0, 0, 1)">三、解题思路</font></h3>

<h4 id="32-攻击步骤"><font style="color: rgba(51, 51, 51, 1)">3.2 攻击步骤</font></h4>
<h5 id="step-1-重现加密过程"><font style="color: rgba(51, 51, 51, 1)">Step 1: 重现加密过程</font></h5>
<p><font style="color: rgba(51, 51, 51, 1)">使用已知的明文数据（user_id=1000, phone=13810000000）重新执行加密流程，得到真实的十六进制密文：</font></p>
<pre><code class="language-python">key = PBKDF2(b"1000", SALT, dkLen=16, count=1000)
cipher = AES.new(key, AES.MODE_CBC, IV)

<h5 id="step-2-建立字符映射表"><font style="color: rgba(51, 51, 51, 1)">Step 2: 建立字符映射表</font></h5>
<p><font style="color: rgba(51, 51, 51, 1)">将真实的十六进制字符串与泄露的混淆结果逐字符对比，建立映射关系：</font></p>
<pre><code class="language-erlang">真实 hex:    a1b2c3d4e5f67890...
混淆结果:    hxnxvjlkjcngzsyc...
           ↓ ↓ ↓ ↓ ↓ ↓ ↓ ↓
映射关系:    a→h, 1→x, b→n, 2→x, c→v, 3→j, ...
</code></pre>
<p><font style="color: rgba(51, 51, 51, 1)">通过这种方式可以完整还原出16个十六进制字符（0-9, a-f）到混淆字符的映射表。</font></p>
<h5 id="step-3-构建逆向映射表"><font style="color: rgba(51, 51, 51, 1)">Step 3: 构建逆向映射表</font></h5>
<p><font style="color: rgba(51, 51, 51, 1)">将正向映射表反转，得到从混淆字符到原始十六进制字符的逆向映射：</font></p>
<pre><code class="language-python">obfuscation_map = {
    '0': 'm', '1': 'n', '2': 'b', '3': 'v', '4': 'c',
    '5': 'x', '6': 'z', '7': 'l', '8': 'k', '9': 'j',
    'a': 'h', 'b': 'g', 'c': 'f', 'd': 'd', 'e': 's', 'f': 'y'
}

reverse_map = {v: k for k, v in obfuscation_map.items()}
</code></pre>
<h5 id="step-4-批量解密"><font style="color: rgba(51, 51, 51, 1)">Step 4: 批量解密</font></h5>
<p><font style="color: rgba(51, 51, 51, 1)">对所有用户数据执行解密流程：</font></p>
<pre><code class="language-plain">混淆密文 → 逆向映射还原 → 十六进制解码 → AES解密 → 去除填充 → 原始手机号
</code></pre>
<h3 id="四解题代码"><font style="color: rgba(0, 0, 0, 1)">四、解题代码</font></h3>
<pre><code class="language-python">import binascii
import csv
from Crypto.Cipher import AES
from Crypto.Protocol.KDF import PBKDF2

# 固定参数
SALT = b"Hidden_Salt_Value"
IV = b"Dynamic_IV_2026!"

def get_key(uid):
    """使用PBKDF2从用户ID生成AES密钥"""
    return PBKDF2(uid, SALT, dkLen=16, count=1000)

def decrypt_data():
    """解密混淆后的手机号数据"""
    # 字符映射表（通过样本分析得出）
    obfuscation_map = {
        '0': 'm', '1': 'n', '2': 'b', '3': 'v', '4': 'c',
        '5': 'x', '6': 'z', '7': 'l', '8': 'k', '9': 'j',
        'a': 'h', 'b': 'g', 'c': 'f', 'd': 'd', 'e': 's', 'f': 'y'
    }
    
    # 构建逆向映射表
    reverse_map = {v: k for k, v in obfuscation_map.items()}
    
    decrypted_data = []
    
    with open("user_data_masked.csv", 'r', encoding='utf-8') as f:
        reader = csv.reader(f)
        next(reader)  # 跳过表头
        
        for row in reader:
            if len(row) != 3:
                continue
            
            user_id, username, masked_phone = row
            
            # 步骤1: 逆向混淆，还原十六进制字符串
            hex_data = ''.join(reverse_map.get(c, c) for c in masked_phone)
            
            # 步骤2: 十六进制解码
            try:
                encrypted_data = binascii.unhexlify(hex_data)
            except Exception as e:
                print(f"解码失败 {user_id}: {e}")
                continue
            
            # 步骤3: AES解密
            try:
                key = get_key(user_id.encode('utf-8'))
                cipher = AES.new(key, AES.MODE_CBC, IV)
                decrypted_padded = cipher.decrypt(encrypted_data)
                
                # 步骤4: 移除PKCS#7填充
                pad_len = decrypted_padded[-1]
                if not (1 &lt;= pad_len &lt;= 16):
                    raise ValueError("无效填充")
                
                decrypted = decrypted_padded[:-pad_len].decode('utf-8')
                decrypted_data.append([user_id, username, decrypted])
                print(f"{user_id:&lt;10} {username:&lt;15} -&gt; {decrypted}")
                
            except Exception as e:
                print(f"解密失败 {user_id}: {e}")
                continue
    
    # 保存结果
    with open("decrypted_phones.csv", 'w', newline='', encoding='utf-8') as f:
        writer = csv.writer(f)
        writer.writerow(["user_id", "username", "phone"])
        writer.writerows(decrypted_data)
    
    print(f"\n✅ 解密完成！共解密 {len(decrypted_data)} 条记录")
    return decrypted_data

if __name__ == "__main__":
    decrypt_data()
</code></pre>
