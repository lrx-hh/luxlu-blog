[复现]数据处理-失灵的日志

一、题目背景

某互联网大厂的安全组件V2.0引入了"双重保护机制"来保护敏感数据：

  1. 第一层防护：使用PBKDF2密钥派生函数结合用户ID生成AES密钥，对手机号进行AES-128-CBC加密
  2. 第二层防护：对加密后的十六进制字符串应用自定义字符映射表进行二次混淆

然而，开发人员在测试环境中遗留了一个包含明文与脱敏结果对照的样本文件 <font style="color:rgb(192, 52, 29);background-color:rgb(246, 246, 246);">sample_leak.txt</font>，这成为了突破口

二、题目分析

2.1 加密流程梳理

完整的加密流程如下：

原始手机号 → PBKDF2派生密钥 → AES-128-CBC加密 → 十六进制编码 → 字符映射混淆 → 最终密文

具体步骤：

  1. 使用 <font style="color:rgb(192, 52, 29);background-color:rgb(246, 246, 246);">PBKDF2(user_id, SALT)</font> 生成16字节AES密钥
  2. 使用AES-128-CBC模式（IV固定为 <font style="color:rgb(192, 52, 29);background-color:rgb(246, 246, 246);">b"Dynamic_IV_2026!"</font>）加密手机号
  3. 将加密结果转换为十六进制字符串
  4. 对十六进制字符串的每个字符应用自定义映射表进行混淆

2.2 已知信息

从题目中获得的关键信息：

User ID: 1000
Original Phone: 13810000000
Masked Result: hxnxvjlkjcngzsycbsjbymygvbfjzjfv

加密参数：

  • 加密算法：AES-128-CBC
  • IV（初始化向量）：<font style="color:rgb(192, 52, 29);background-color:rgb(246, 246, 246);">b"Dynamic_IV_2026!"</font>
  • 盐值：<font style="color:rgb(192, 52, 29);background-color:rgb(246, 246, 246);">b"Hidden_Salt_Value"</font>（需要通过分析推断）
  • PBKDF2迭代次数：1000（标准配置）

三、解题思路

关键在于利用已知的明文-密文对来逆向推导字符映射表。

3.2 攻击步骤

Step 1: 重现加密过程

使用已知的明文数据（user_id=1000, phone=13810000000）重新执行加密流程，得到真实的十六进制密文：

key = PBKDF2(b"1000", SALT, dkLen=16, count=1000)
cipher = AES.new(key, AES.MODE_CBC, IV)
encrypted = cipher.encrypt(pad(b"13810000000"))
real_hex = encrypted.hex()  # 得到真实的十六进制字符串
Step 2: 建立字符映射表

将真实的十六进制字符串与泄露的混淆结果逐字符对比，建立映射关系：

真实 hex:    a1b2c3d4e5f67890...
混淆结果:    hxnxvjlkjcngzsyc...
           ↓ ↓ ↓ ↓ ↓ ↓ ↓ ↓
映射关系:    a→h, 1→x, b→n, 2→x, c→v, 3→j, ...

通过这种方式可以完整还原出16个十六进制字符（0-9, a-f）到混淆字符的映射表。

Step 3: 构建逆向映射表

将正向映射表反转，得到从混淆字符到原始十六进制字符的逆向映射：

obfuscation_map = {
    '0': 'm', '1': 'n', '2': 'b', '3': 'v', '4': 'c',
    '5': 'x', '6': 'z', '7': 'l', '8': 'k', '9': 'j',
    'a': 'h', 'b': 'g', 'c': 'f', 'd': 'd', 'e': 's', 'f': 'y'
}


reverse_map = {v: k for k, v in obfuscation_map.items()}

Step 4: 批量解密

对所有用户数据执行解密流程：

混淆密文 → 逆向映射还原 → 十六进制解码 → AES解密 → 去除填充 → 原始手机号

四、解题代码

import binascii
import csv
from Crypto.Cipher import AES
from Crypto.Protocol.KDF import PBKDF2


固定参数
SALT = b”Hidden_Salt_Value”
IV = b”Dynamic_IV_2026!”


def get_key(uid):
    “””使用PBKDF2从用户ID生成AES密钥”””
    return PBKDF2(uid, SALT, dkLen=16, count=1000)


def decrypt_data():
    “””解密混淆后的手机号数据”””
    # 字符映射表（通过样本分析得出）
    obfuscation_map = {
        ‘0’: ‘m’, ‘1’: ‘n’, ‘2’: ‘b’, ‘3’: ‘v’, ‘4’: ‘c’,
        ‘5’: ‘x’, ‘6’: ‘z’, ‘7’: ‘l’, ‘8’: ‘k’, ‘9’: ‘j’,
        ‘a’: ‘h’, ‘b’: ‘g’, ‘c’: ‘f’, ‘d’: ‘d’, ‘e’: ‘s’, ‘f’: ‘y’
    }


# 构建逆向映射表
reverse_map = {v: k for k, v in obfuscation_map.items()}

decrypted_data = []

with open("user_data_masked.csv", 'r', encoding='utf-8') as f:
    reader = csv.reader(f)
    next(reader)  # 跳过表头
    
    for row in reader:
        if len(row) != 3:
            continue
        
        user_id, username, masked_phone = row
        
        # 步骤1: 逆向混淆，还原十六进制字符串
        hex_data = ''.join(reverse_map.get(c, c) for c in masked_phone)
        
        # 步骤2: 十六进制解码
        try:
            encrypted_data = binascii.unhexlify(hex_data)
        except Exception as e:
            print(f"解码失败 {user_id}: {e}")
            continue
        
        # 步骤3: AES解密
        try:
            key = get_key(user_id.encode('utf-8'))
            cipher = AES.new(key, AES.MODE_CBC, IV)
            decrypted_padded = cipher.decrypt(encrypted_data)
            
            # 步骤4: 移除PKCS#7填充
            pad_len = decrypted_padded[-1]
            if not (1 &lt;= pad_len &lt;= 16):
                raise ValueError("无效填充")
            
            decrypted = decrypted_padded[:-pad_len].decode('utf-8')
            decrypted_data.append([user_id, username, decrypted])
            print(f"{user_id:&lt;10} {username:&lt;15} -&gt; {decrypted}")
            
        except Exception as e:
            print(f"解密失败 {user_id}: {e}")
            continue

# 保存结果
with open("decrypted_phones.csv", 'w', newline='', encoding='utf-8') as f:
    writer = csv.writer(f)
    writer.writerow(["user_id", "username", "phone"])
    writer.writerows(decrypted_data)

print(f"\n✅ 解密完成！共解密 {len(decrypted_data)} 条记录")
return decrypted_data

def step(x, y, p):
    return (5y + 7) % p, (11x + 13) % p


def apply_steps(x, y, n, p):
    for _ in range(n):
        x, y = step(x, y, p)
    return x, y


Compute alpha, beta for x10 = alpha * x0 + beta (when y0=0)
x0_0, y0_0 = 0, 0

x0_1, y0_1 = 1, 0
x10_1, _ = apply_steps(x0_1, y0_1, 10, p)

beta = x10_0 % p


Compute gamma, delta for y10 = gamma * y0 + delta (when x0=0)
_, y10_0 = apply_steps(0, 0, 10, p)
_, y10_1 = apply_steps(0, 1, 10, p)

for s0 in roots0:
    for s1 in roots1:
        # Solve: (alphau + beta)(gamma*(s0/u) + delta) = s1
        # => (alphau + beta)(gammas0 + deltau) = s1 * u
        # => alphadeltau^2 + (alphagammas0 + betadelta - s1)u + betagammas0 = 0
        A = (alpha * delta) % p
        B = (alpha * gamma * s0 + beta * delta - s1) % p
        C = (beta * gamma * s0) % p


    # Solve A*u^2 + B*u + C = 0 mod p
    if A == 0:
        if B == 0:
            continue
        u = (-C * pow(B, -1, p)) % p
        us = [u]
    else:
        # discriminant
        D = (B*B - 4*A*C) % p
