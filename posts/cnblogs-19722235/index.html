关于内存取证

什么是内存镜像（.vmem）？

.vmem 是 VMware 虚拟机运行时的内存快照文件，相当于把电脑 RAM 里的所有内容完整保存成一个文件。通过分析它，我们可以还原出案发时系统的几乎所有状态：运行的程序、网络连接、注册表、甚至密码。

内存取证工具Volatility

Volatility 是内存取证领域最主流的开源工具，能从内存镜像中提取各类信息。

使用 Volatility 3（命令为 vol.exe）即可

Step3:获取 PC 名

Windows 的计算机名存储在注册表的固定路径下：

HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Control\ComputerName\ComputerName

Windows 注册表是一个层级数据库，存储系统和软件的几乎所有配置。

ControlSet001 是活跃的控制集（类似"当前配置"）

:::color1
Windows 的注册表（Registry）是一个树形数据库，HKEY_LOCAL_MACHINE\SYSTEM 分支下保存着操作系统级别的配置。计算机名这类"机器身份"信息由系统在安装时写入，后续可以通过"系统属性"界面修改，修改后依然写回到这个注册表路径。

Volatility 的 printkey 插件直接从内存中解析注册表蜂巢（hive）文件的内存映射，因此即使没有硬盘镜像，只要内存里有注册表的缓存，就能读出这些值。

:::

Volatility 3 的 windows.registry.printkey 插件可以直接读取注册表内容：

:::warning

flag包含两部分

  1. 游戏名称 → 从进程列表里找正在运行的游戏程序
  2. 服务器 IP → 从网络连接里找该游戏进程连接的远端地址

插件windows.pstree/netscan/netstat

插件 功能
windows.pstree 查看所有运行中的进程（像任务管理器）
windows.netscan 池扫描（Pool Scanning），在内存中搜索网络对象的特征标记
扫描内存中的网络连接结构体，包括已关闭的连接
已关闭（CLOSED）的连接大多数也能找到
速度较慢，但是扫描的相对比较全面
windows.netstat 遍历系统维护的活跃连接链表
只显示活跃连接
已关闭（CLOSED）的连接可能遗漏
速度很快，但是有很多遗漏

为什么用 netscan 而不是 netstat？
netstat只显示当前活跃连接，游戏连接可能已断开（状态为 CLOSED）

题干说是账号登录的频道名是 Lunar-3

游戏登录时，客户端会把频道名和账号名同时保存在内存里，从而用于界面显示、数据请求等。因此只要在游戏进程的内存中找到 Lunar-3，看它附近存着什么字符串，账号名就藏在那里。

为什么账号名会明文存在内存里？

游戏客户端在登录后需要实时显示账号名（界面 UI、角色选择页面等），因此账号名会以明文字符串形式存在进程的堆内存（heap）或数据段中。除非游戏客户端专门做了内存保护（如加密存储），否则只要进程在运行，账号名就可以被读出来。这也是游戏外挂/内存编辑工具能读取游戏数据的根本原因。

什么是进程内存Dump？

Volatility 可以把某个进程占用的全部虚拟内存导出成一个文件（.dmp），之后就可以像普通文件一样用 Python、strings 等工具搜索里面的内容，不需要反复调用 Volatility。

字节签名（Byte Signature）

上一题用"关键词附近搜索"找到了账号名，但角色名没有像Lunar-3这样方便的文本线索可以利用。

题目给了另一种方法：字节签名定位。通过逆向分析或内存对比，研究人员发现：MapleStory客户端在内存中存储已登录角色名时，角色名之前总是有一段固定的字节模式。只要找到这个模式，紧随其后的字符串就是角色名。

0x64  0x??{6-8}  0x40 0x06  0x??{18}  0x5a 0x0c  0x00 0x00
部分 含义
0x64 固定字节，值为 100（十进制）
0x??{6-8} 6 到 8 个任意字节（通配符）
0x40 0x06 固定两字节
0x??{18} 18 个任意字节（通配符）
0x5a 0x0c 固定两字节
0x00 0x00 两个零字节（通常是字段分隔符或对齐填充）

签名结束后，紧接着就是角色名字符串

Step1:确认进程 dump 已存在

dumps/pid.708.dmp   （LunarMS.exe，PID 708）

如果还没有导出，先执行：

vol.exe -f OtterCTF.vmem --output-dir ./dumps windows.memmap --pid 708 --dump

Step2:将字节签名转换为正则表达式

Python 的 re 模块支持二进制正则，把签名翻译如下：

签名部分 Python 正则
0x64 \x64
0x??{6-8} .{6,8}
0x40 0x06 \x40\x06
0x??{18} .{18}
0x5a 0x0c \x5a\x0c
0x00 0x00 \x00\x00 
pattern = re.compile(b'\x64.{6,8}\x40\x06.{18}\x5a\x0c\x00\x00', re.DOTALL)

re.DOTALL 标志使 . 能匹配所有字节包括 \x00（默认情况下 . 不匹配换行符 \n 即 \x0a，二进制搜索必须加这个标志）

vol2 clipborad插件的原理

  1. 找到 win32k.sys 在内核中的基地址
  2. 定位 tagWINDOWSTATION 结构（记录了所有窗口站）
  3. 遍历每个窗口站的 tagCLIPDATA 链表
  4. 对每个 CF_UNICODETEXT 条目，读取 cbData 字节的数据
  5. 以 UTF-16LE 解码，输出明文

为什么剪贴板内容在内存里？

Windows 剪贴板不会自动清除，只有以下情况才会清空：

  1. 新内容写入剪贴板（下一次 Ctrl+C）
  2. 程序主动调用 EmptyClipboard()
  3. 系统重启

内存快照时Rick可能刚复制完还没粘贴或者粘贴完程序没有清除剪贴板。

这样这样Windows剪切板数据在有新内容覆盖前始终保留在内存中。

因为内存快照捕获的是某一瞬间的内存状态。所以，时机合适就能捕获到剪切板内容
